    Администрирование отеля (Django Rest Framework)

    Текст технического задания

    Создать программную систему, предназначенную для администратора гостиницы.

    Такая система должна обеспечивать хранение сведений об имеющихся в гостинице номерах, о проживающих в гостинице клиентах и о служащих, убирающихся в номерах. Количество номеров в гостинице известно, и имеются номера трех типов: одноместный, двухместный и трехместный, отличающиеся стоимостью проживания в сутки. В каждом номере есть телефон.

    О каждом проживающем должна храниться следующая информация: номер паспорта, фамилия, имя, отчество, город, из которого он прибыл, дата поселения в гостинице, выделенный гостиничный номер.

    О служащих гостиницы должна быть известна информация следующего содержания: фамилия, имя, отчество, где (этаж) и когда (день недели) он убирает. Служащий гостиницы убирает все номера на одном этаже в определенные дни недели, при этом в разные дни он может убирать разные этажи.

    Дополнительные требования

    Работа с системой предполагает получение следующей информации:

    • о клиентах, проживавших в заданном номере, в заданный период времени;
    • о количестве клиентов, прибывших из заданного города,
    • о том, кто из служащих убирал номер указанного клиента в заданный день недели,
    • сколько в гостинице свободных номеров;
    • список клиентов с указанием места жительства, которые проживали в те же дни, что и заданный клиент, в определенный период времени.

    Администратор должен иметь возможность выполнить следующие операции:

    • принять на работу или уволить служащего гостиницы;
    • изменить расписание работы служащего;
    • поселить или выселить клиента.

    Необходимо предусмотреть также возможность автоматической выдачи отчета о работе гостиницы за указанный квартал текущего года. Такой отчет должен содержать следующие сведения:

    • число клиентов за указанный период в каждом номере;
    • количество номеров на каждом этаже;
    • общая сумма дохода за каждый номер;
    • суммарный доход по всей гостинице.

    В процессе работы использовались следующие технологии:

    • Django 3, Django REST Framework для программирования логики работы API;
    • DjangoORM для реализации модели базы данных;
    • Djoser в целях регистрации / авторизации по токенам / вывода информации о текущем пользователе средствами Djoser.
    • CORS (Cross-origin resource sharing) во имя следующего этапа проекта.

    Теперь же в работе сервиса используются сериализаторы, которые позволяют преобразовывать сложные данные, такие как наборы запросов и экземпляры моделей, в собственные типы данных Python, и затем могут быть легко преобразованы в JSON.

    Описание моделей и endpoint-ов

    Комната Room

    Особенность: использование Enum (в данном контексте - Choices) для выбора вида комнаты. fk этажа для выполнения дополнительных функциональных требований.

    CREATE TABLE `room` (
  `id` int NOT NULL AUTO_INCREMENT,
  `hotel_id` int NOT NULL,
  `room_type` varchar(6) NOT NULL DEFAULT 'Single',
  `price` int unsigned NOT NULL,
  `phone_num` int NOT NULL,
  `floor_num` smallint NOT NULL,
  PRIMARY KEY (`id`),
  KEY `room_hotel_id` (`hotel_id`),
  KEY `room_floor_num` (`floor_num`),
  CONSTRAINT `room_hotel_id` FOREIGN KEY (`hotel_id`) REFERENCES `hotel` (`id`) ON DELETE CASCADE,
  CONSTRAINT `room_floor_num` FOREIGN KEY (`floor_num`) REFERENCES `floor` (`floor_num`) ON DELETE CASCADE,
  CONSTRAINT `room_type_check` CHECK (`room_type` in ('Single','Double','Triple'))
);

    Получить информацию о всех комнатах отеля с возможностью добавления новой сущности

    GET, POST /hotel/rooms/

    Получить информацию об одной комнате с возможностью изменения и удаления объекта

    GET, PATCH, DELETE /hotel/rooms/<room_id>/

    Постоялец Client

    CREATE TABLE `client` (
  `id` int NOT NULL AUTO_INCREMENT,
  `passport_number` varchar(11) NOT NULL,
  `last_name` varchar(30) NOT NULL,
  `first_name` varchar(30) NOT NULL,
  `father_name` varchar(30) DEFAULT NULL,
  `origin_city` varchar(50) NOT NULL,
  PRIMARY KEY (`id`)
);

    Получить информацию о всех клиентах отеля с возможностью добавления нового

    GET, POST /hotel/clients/

    Получить информацию об одном клиенте с возможностью изменения и удаления объекта

    GET, PATCH, DELETE /hotel/clients/<client_id>/

    Работник Employee

    CREATE TABLE `employee` (
  `id` int NOT NULL AUTO_INCREMENT,
  `hotel_id` int NOT NULL,
  `last_name` varchar(30) NOT NULL,
  `first_name` varchar(30) NOT NULL,
  `father_name` varchar(30) DEFAULT NULL,
  `is_employed` tinyint(1) NOT NULL DEFAULT '1',
  PRIMARY KEY (`id`),
  KEY `employee_hotel_id` (`hotel_id`),
  CONSTRAINT `employee_hotel_id` FOREIGN KEY (`hotel_id`) REFERENCES `hotel` (`id`) ON DELETE CASCADE
);

    Получить информацию о всех работниках отеля с возможностью добавления нового

    GET, POST /hotel/employees/

    Получить информацию об одном работнике с возможностью изменения и удаления объекта

    GET, PATCH, DELETE /hotel/employees/<employee_id>/

    Бронирование Booking

    Внешние ключи Клиента и Комнаты обеспечивают связность и логичность базы данных.

    CREATE TABLE `booking` (
  `id` int NOT NULL AUTO_INCREMENT,
  `client_id` int NOT NULL,
  `room_id` int NOT NULL,
  `start_date` date NOT NULL,
  `end_date` date NOT NULL,
  `is_verified` tinyint(1) NOT NULL DEFAULT '0',
  PRIMARY KEY (`id`),
  KEY `booking_client_id` (`client_id`),
  KEY `booking_room_id` (`room_id`),
  CONSTRAINT `booking_client_id` FOREIGN KEY (`client_id`) REFERENCES `client` (`id`) ON DELETE CASCADE,
  CONSTRAINT `booking_room_id` FOREIGN KEY (`room_id`) REFERENCES `room` (`id`) ON DELETE CASCADE
);

    Получить информацию о всех бронированиях в отеле с возможностью добавления нового

    GET, POST /hotel/bookings/

    Получить информацию об одном бронировании с возможностью изменения и удаления объекта

    GET, PATCH, DELETE /hotel/bookings/<booking_id>/

    Уборка Cleaning

    Связана с работником, проводящим уборку, и комнатой, в которой совершилось преступление.

    CREATE TABLE `cleaning` (
  `id` int NOT NULL AUTO_INCREMENT,
  `floor_num` smallint NOT NULL,
  `employee_id` int NOT NULL,
  `week_day` varchar(3) NOT NULL,
  PRIMARY KEY (`id`),
  KEY `cleaning_floor_num` (`floor_num`),
  KEY `cleaning_employee_id` (`employee_id`),
  CONSTRAINT `cleaning_employee_id` FOREIGN KEY (`employee_id`) REFERENCES `employee` (`id`) ON DELETE CASCADE,
  CONSTRAINT `cleaning_floor_num` FOREIGN KEY (`floor_num`) REFERENCES `floor` (`floor_num`) ON DELETE CASCADE,
  CONSTRAINT `week_day_check` CHECK (`week_day` in ('Mon','Tue','Wed','Thu','Fri','Sat','Sun'))
);

    Получить информацию о всех уборках комнат с возможностью добавления нового

    GET, POST /hotel/cleanings/

    Получить информацию об одной уборки комнат с возможностью изменения и удаления объекта

    GET, PATCH, DELETE /hotel/cleanings/<cleaning_id>/

    Описание дополнительных endpoint-ов

    Получение информации:

    • о клиентах, проживавших в заданном номере, в заданный период времени

    GET /hotel/clients-in-room/<room_id>/<start_date>/<end_date>/

    • о количестве клиентов, прибывших из заданного города

    GET /hotel/clients-from-city/<origin_city>/

    • о том, кто из служащих убирал номер указанного клиента в заданный день недели

    GET /hotel/cleaning-employees/<client_id>/<week_day>/

    • о количестве в гостинице свободных номеров

    GET /hotel/free-rooms/<start_date>/<end_date>/

    • о списке клиентов с указанием места жительства, которые проживали в те же дни, что и заданный клиент

    GET /hotel/clients-while-client/<request_client_id>/

    Выдача отчёта за запрашиваемый квартал

    Отчёт содержит следующие данные:

    • число клиентов за указанный период в каждом номере;
    • количество номеров на каждом этаже;
    • общая сумма дохода за каждый номер;
    • суммарный доход по всей гостинице.

    GET /hotel/report/<requested_quarter>/
