    Администрирование отеля (Vue.js)

    Продолжение работы над сервисом управления гостиницей с акцентом на расширение с помощью Vue.js - JavaScript-фреймворка, предназначенного для создания пользовательских интерфейсов и одностраничных приложений.

    На рисунке ниже представлена неполная UML Use Case диаграмма сервиса. Неполная оа по той причине, что только сущность Клиента была раскрыта со связями include, но в моей интуитивно понятной работе очевидно, что в других сущностях с такими методами присутствуют такие же включения.

    Работа Vue.js

    Интерфейсы авторизации и регистрации

    На протяжении работы запросы выполнялись с помощью axios:

    import axios from "axios";

const instance = axios.create({
    baseURL: 'http://127.0.0.1:8000/',
    timeout: 1000,
});

export default instance

    Axios — клиентская библиотека HTTP, которая позволяет отправлять запросы (реквесты) к заданной конечной точке. В моём случае это был сервис, логику которого прописывалась в предыдущей работе.

    Первое вхождение на сайт - обработка состояния авторизации пользователя. Доступ к функционалу сайта открывается только после регистрации и авторизации:

    Register page

    POST /register - это форма с полями для имени пользователя, электронной почты и пароля. Активная ссылка под кнопкой переносит пользователя на Login page.

    Login page

    POST /login - имеет поля для имени пользователя и его пароля. Ссылка под кнопкой логин переадресовывает пользователя на Register page. При успешном логине пользователю присуждается токен, который хранится в базе данных приложения.

    Ту же самую переадресацию делает страница при некорректно введённых данных, которые отсутствуют в системе.

    Интерфейсы моделей (списки, страницы редактирования и добавления новых объектов)

    Rooms Page

    GET /rooms

    Таким образом встречает сайт успешно залогиненного пользователя. Страница со всеми комнатами отеля состоит из двух столбцов из карточек, в каждой карточке представлена информация об объекте, есть кнопки редактирования (redirect PATCH /rooms/:id) и удаления.

    function getRooms() {
  instance.get('/hotel/rooms/').then(
    response => {
      if (response.status === 200) {
        rooms.value = response.data
      }
    }
  ).catch(error => console.log(error))
}

    При попытке удаления высвечивает окно для подтверждения действия (аналогично оно высвечивается при попытке удаления всех других сущностей):

    function deleteRoom(id) {
  if (confirm("Are you sure you want to delete this room?")) {
    instance.delete(`/hotel/rooms/${id}/`).then(response => {
        if (response.status === 204) {
          getRooms()
        }
      }
    ).catch(error => console.log(error))
  } else {
    console.log("Room deletion cancelled.")
  }
}

    Clients Page

    GET /clients

    Тот же карточный вид с соответствующими кнопка в каждой. Однако под заголовок добавились две дополнительные опции: добавление и сортировка.

    Форма для добавления POST /clients/add имеет поля для создания нового клиента.

    Страница сортировки GET /clients/sort представляет собой v-data-table с встроенным поиском по таблице, сортировке по каждому столбцу.

    Employees Page

    GET /employees

    Работники отеля несут немного информации, поэтому уменьшенный масштаб и три столбца. Карточки, кнопки - по классике. Сортировать их не было смысла, поэтому есть только одна кнопка для добавления сущности POST /employees/add.

    HTML-код добавления нового работника отеля:

    Bookings Page

    GET /bookings

    Страница с бронированиями уже интереснее за счёт наличия внешних ключей.

    В формах для редактирования PATCH /bookings/:id и добавления POST /bookings/add присутствуют select-поля, состоящие из доступных внешних ключей. Для бронирования это клиенты и комнаты.

    function getPage() {
  instance.get(`/hotel/bookings/${router.currentRoute.value.params.id}/`)
    .then(response => {
        if (response.status === 200) {
          form.value = response.data
        }
      }
    ).catch(error => console.log(error))

  instance.get('hotel/rooms/').then(response => {
      if (response.status === 200) {
        room_select.value = response.data
      }
    }
  ).catch(error => console.log(error))

  instance.get('hotel/clients/').then(response => {
      if (response.status === 200) {
        client_select.value = response.data
      }
    }
  ).catch(error => console.log(error))
}

    Quarter Report Page

    GET /report

    Завершающая страница нашего обзора - отчёты по работе отеля за определённую четверть года. Два select-поля - выбор четверти и интересующей части отчёта.

    function fetchReport() {
  if (selectedQuarter.value && selectedPart.value) {
    loading.value = true;
    instance
      .get(`/hotel/report/${selectedQuarter.value}/`)
      .then((response) => {
        reportData.value = response.data[selectedPart.value];
      })
      .catch((error) => {
        console.error("Error fetching report:", error);
        reportData.value = "Failed to load report.";
      })
      .finally(() => {
        loading.value = false;
      });
  }
}

watch([selectedQuarter, selectedPart], fetchReport);
