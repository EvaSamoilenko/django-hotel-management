    Список Отелей (фреймворк Django)

    Функциональные требования по ТЗ:

    • Необходимо учитывать название отеля, владельца отеля, адрес, описание, типы номеров, стоимость, вместимость, удобства.
    • Регистрация новых пользователей.
    • Просмотр и резервирование номеров. Пользователь должен иметь возможность редактирования и удаления своих резервирований.
    • Написание отзывов к номерам. При добавлении комментариев, должны сохраняться период проживания, текст комментария, рейтинг (1-10), информация о комментаторе.
    • Администратор должен иметь возможность заселить пользователя в отель и выселить из отеля средствами Django-admin.
    • В клиентской части должна формироваться таблица, отображающая постояльцев отеля за последний месяц.

    Основная логика сериализации, отображения ответов и навигации между страницами происходит в .html файлах проекта, которые хранятся в директории hotel_web_project/templates.

    Тем самым в результате разработки были изучены не только способы программирования с помощью фреймворка Django 3 (структура проекта, функциональные и классовые представления, редиректы и построения моделей), но и варианты работы с HTML, bootstrap, их внутренними обработчиками и сериализаторами.

    Большое количество методов зависят от состояния авторизации пользователя: если авторизация не производилась, то методы будут недоступны.

    Результаты работы

    Описание

    Приложение Hotel App умеет регистрировать нового пользователя и позволяет ему зайти в свой аккаунт. Во время своего сеанса пользователь может рассматривать список доступных отелей, комнаты в этих отелях и отзывы на них. Также юзер способен просматривать свои бронирования, создавать новые, изменять и удалять их. Пользователь может свободно выражать свои мысли о проживании в разделе отзывов.

    Модели

    • Hotel: название, владелец, адрес и описание:
    CREATE TABLE "hotel" (
    "id" serial NOT NULL PRIMARY KEY,
    "name" varchar(50) NOT NULL,               -- название отеля
    "owner" varchar(30) NOT NULL,              -- владелец отеля
    "address" varchar(100) NOT NULL,           -- адрес отеля
    "description" varchar(100) NOT NULL        -- описание отеля
);
    • Room: fk отель, тип комнаты, цена, вместительность, дополнительные атрибуты:
    CREATE TABLE "room" (
    "id" serial NOT NULL PRIMARY KEY,
    "hotel_id" integer NOT NULL,               -- FK на отель
    "room_type" varchar(17) NOT NULL           -- тип комнаты (Standard Single, Double и т.д.)
        CHECK (
            "room_type" IN (
                'Standard Single',
                'Standard Double',
                'Standard Twin',
                'Quadruple Room',
                'Presidential Suit'
            )
        ),
    "price" integer NOT NULL                   -- цена за ночь (только положительные числа)
        CHECK ("price" >= 0),
    "capacity" integer NOT NULL                -- вместительность (1-5 человек)
        CHECK ("capacity" BETWEEN 1 AND 5),
    "description" varchar(100) NULL,           -- дополнительное описание (может быть пустым)

    FOREIGN KEY ("hotel_id") REFERENCES "hotel" ("id") ON DELETE CASCADE
);
    • Booking: fk комната, fk клиента, даты бронирования:
    CREATE TABLE "booking" (
    "id" serial NOT NULL PRIMARY KEY,
    "user_id" integer NOT NULL,                -- FK на пользователя (клиента)
    "room_id" integer NOT NULL,                -- FK на комнату
    "start_date" date NOT NULL,                -- дата заезда
    "end_date" date NOT NULL,                  -- дата выезда

    FOREIGN KEY ("user_id") REFERENCES "auth_user" ("id") ON DELETE CASCADE,
    FOREIGN KEY ("room_id") REFERENCES "room" ("id") ON DELETE CASCADE,

    CHECK ("end_date" > "start_date")          -- проверка, что дата выезда позже заезда
);
    • Review: fk бронирование, рейтинг, отзыв:
    CREATE TABLE "review" (
    "id" serial NOT NULL PRIMARY KEY,
    "booking_id" integer NOT NULL,             -- FK на бронирование
    "rating" smallint NOT NULL                 -- рейтинг (например, 1-5 звёзд)
        CHECK ("rating" BETWEEN 1 AND 5),
    "comment" text NULL,                       -- текстовый отзыв (может быть пустым)

    FOREIGN KEY ("booking_id") REFERENCES "booking" ("id") ON DELETE CASCADE
);

    Пример структуры модели в перевод на язык django.db.models:

    class Booking(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    room = models.ForeignKey(Room, on_delete=models.CASCADE)
    start_date = models.DateField()
    end_date = models.DateField()

    Функционал

    Регистрация и авторизация пользователя

    Регистрация

    POST /signup/

    Создание нового аккаунта пользователя.

    Параметры запроса (тело JSON):

    Поле Тип Обязательность Описание
    username string Да Логин пользователя
    password1 string Да Пароль
    password2 string Да Подтверждение пароля

    Аутентификация и авторизация

    POST /login/

    Проверка наличия профиля пользователя в базе данных и вход на главную страницу сайта.

    Параметры запроса:

    Поле Тип Обязательность Описание
    username string Да Логин пользователя
    password string Да Пароль

    Просмотр списка отелей

    GET /hotels/

    Получение главной страницы сайта со списком доступных для бронирования отелей.

    Особенность: классовое представление ListView

    class HotelListView(ListView):
    model = Hotel
    context_object_name = 'hotels'
    template_name = 'hotel/hotels_list.html'

    Доступные комнаты отеля

    GET /hotels/{hotel_id}/

    Получение списка комнат заданного отеля с соответствующими характеристиками комнаты и возможностями (кнопками) последующего взаимодействия.

    def rooms_list(request, hotel_id):
    rooms = Room.objects.filter(hotel_id=hotel_id)
    return render(request, "hotel/hotel_rooms.html", {"rooms": rooms})

    Бронирование номера

    POST /hotels/{hotel_id}/{room_id}/

    Бронирование выбранного номера в заданном отеле.

    Параметры запроса:

    Поле Тип Обязательность Описание
    hotel_id int Да id гостиницы
    room_id int Да id выбранного номера
    start_date datetime Да дата заезда в номер
    end_date datetime Да дата выезда из номера

    Отзывы

    GET /hotels/{hotel_id}/{room_id}/reviews/

    Просмотр отзывов к запрашиваемой комнате.

    Параметры запроса:

    Поле Тип Обязательность Описание
    hotel_id int Да id гостиницы
    room_id int Да id выбранного номера 
    @login_required(login_url='/login/')
def book_room(request, hotel_id, room_id):
    room = Room.objects.get(id=room_id)
    user = request.user
    context = {}
    form = BookingCreateForm(request.POST or None)
    if form.is_valid():
        booking = form.save(commit=False)
        booking.user = user
        booking.room = room
        form.save()
        return redirect("bookings")
    context['form'] = form
    return render(request, "booking/book_room.html", context)

    Личные бронирования пользователя

    GET /bookings/

    Получение списка бронирований номеров конкретного пользователя.

    Особенность: функциональное представление; фильтрация в базе данных происходит на основе user, под именем которого залогинен пользователь на сайте.

    def user_bookings(request):
    bookings = Booking.objects.filter(user_id=request.user)
    return render(request, "booking/user_bookings.html", {"bookings": bookings})

    Написать отзыв

    POST /bookings/{booking_id}/write_review

    Отправить в базу данных отзыв на прошедшее бронирование пользователя.

    Поле Тип Обязательность Описание
    booking_id int Да номер бронирования
    review string Да письменный отзыв
    rate int Да оценка (1-10)

    Изменить параметры бронирования

    PATCH /bookings/{pk}/edit/

    Внос изменений в бронирование, которое ещё не наступило, то есть доступно для обновления.

    Поле Тип Обязательность Описание
    booking_id int Да номер бронирования
    start_date datetime Нет дата заезда в номер
    end_date datetime Нет дата выезда из номера 
    class BookingUpdateView(UpdateView):
    model = Booking
    fields = ['start_date', 'end_date']
    success_url = '/bookings/'
    template_name = 'booking/booking_update.html'

    Статистика за прошедший месяц

    GET hotels/{hotel_id}/monthly/

    Получение статистических данных отеля за прошедший месяц.

    Особенность: страница доступна только тем пользователям, которые обладают правами администратора. При попытке данного запроса неавторизарованным пользователем будет возвращено сообщение "You have no right to be here! (Be a staff member to see this page)".

    def monthly_clients(request, hotel_id):
    last_month = datetime.now() - timedelta(days=30)
    bookings = Booking.objects.filter(start_date__gt=last_month).filter(room__hotel_id=hotel_id)
    return render(request, "hotel/monthly_clients.html", {"bookings_month": bookings})
